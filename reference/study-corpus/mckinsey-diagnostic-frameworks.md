# McKinsey diagnostic frameworks (synthesized)

Sources: internal diagnostic playbooks extracted to `reference/study-corpus/extracted/` — procurement EM guide, B2B Commercial Diagnostics Desk, eCompass DTC, ecosystem diagnostics, corporate legal CBF, wagon operator holistic diagnostic, municipal finance, philanthropy ECD, supply chain sustainability, oncology trends.

Used to ground **Agent 4 — Improvement Initiatives** (process-driven, not brainstorming).

---

## Universal diagnostic mental model

McKinsey diagnostics consistently follow a **two-lens** structure:

| Lens | Question | Typical artifacts |
|------|----------|-------------------|
| **Value at stake** | How much is realistically available? | Benchmark gap, sizing tree, contract teardown, peer percentile |
| **Operating effectiveness** | Why is value leaking today? | Process map, maturity assessment, pain points, root causes |

Then: **synthesize → prioritize → time-phase → hand off to implementation**.

### Three-stage flow (wagon operator, procurement, commercial)

1. **Current state analysis** — data gathering, interviews, benchmarking (controlling for comparability)
2. **Priority areas & quick wins** — align list of improvements with expected impact
3. **Full potential estimate** — triangulate top-down and bottom-up; syndicate with client

### What diagnostics are / are not

- **Are:** repeatable structure, team judgment accelerators, fact-base builders, modular asset menus
- **Are not:** one-size-fits-all scorecards, guaranteed savings, client-ready without synthesis, tech roadmaps by default

## Core vocabulary (jargon)

| Term | Meaning in practice |
|------|---------------------|
| **IDP** | Initial Diagnostic Phase — deep inside-out assessment |
| **LOP** | Letter of Proposal — often preceded by rapid outside-in diagnostic |
| **Fact base / fact pack** | Quantified, sourced evidence for leadership decisions |
| **Value at stake** | Directional or sized opportunity (growth, margin, savings, risk) |
| **Addressable spend** | Normalized scope with clear exclusions — avoid scope creep |
| **Value blocker** | Finding that materially prevents capture of value — prioritize over noise |
| **Efficiency improvement** | Reduces effort/cycle time without direct P&L line — still valuable |
| **Enablers** | Capabilities that scale the process (tech/data, governance, KPIs, talent, category mgmt) |
| **Maturity** | Reactive → Inconsistent → Institutionalized (simple G/Y/R scale) |
| **Current vs target state** | Gap analysis per process step and enabler |
| **Root cause** | Why procurement/commercial/ops cannot capture value consistently |
| **Triangulation** | Top-down benchmark + bottom-up validation + client syndication |
| **Run-rate vs in-year** | Timing realism for savings; avoid double-counting renewals |
| **Strike team** | Cross-functional squad for high-value category/RFP work |
| **Rewired Growth** | Commercial excellence: right opportunity, people/GTM, offer, price, every time |
| **EPR** | Capability questionnaire style assessment (ecosystem fit) |

## Initiative generation principles (Agent 4)

1. **Process-first** — every initiative maps to process step(s) and pain point(s); no orphan ideas
2. **Separate value blockers from efficiency** — procurement: "which enablers are true value blockers vs efficiency improvements?"
3. **Lever taxonomy** — process fix, AI/automation, tech/data foundation, policy/governance, org design
4. **Horizon phasing** — H1 near-term/quick wins, H2 scale/mid-term, H3 structural/transform
5. **Prioritization criteria** (procurement synthesis):
   - Near-term value impact
   - Timing dependency (renewals, RFPs, hiring cycles)
   - Execution complexity (1–2 months vs 12+ months)
   - Capacity constraints (can the org absorb change?)
6. **Lifecycle awareness** — flag in-flight, partially existing, net-new; detect duplicates/overlaps
7. **Evidence strength** — interview, document, benchmark, or inference; never present inference as fact
8. **Savings vs operational value** — not all improvements are P&L savings; label value type honestly

## Modular building blocks (reused across domains)

### Process + enablers (procurement S2P, eCompass journey)

- End-to-end **process steps** (8-stage S2P, customer journey, legal matter lifecycle)
- Parallel **enabler assessment** (tech/data, org/governance, KPIs, talent, category strategy)

### Commercial excellence (B2B Diagnostics Desk)

- Productivity, efficiency, coverage levers
- Activity/time analytics, pipeline health, capability maturity (CCAT)
- Growth at stake vs margin at stake — separate sizing lenses

### Benchmarking discipline

- Peer set with revenue/industry normalization
- Gap to next quartile / percentile — directional for prioritization
- Limitations explicit (structural differences, data granularity)

## Output shape for downstream agents

Initiatives should be ready for **value modeling**, **roadmapping**, and **future-state workflow** agents:

- Normalized IDs linking workflow → process_step → pain_point → initiative
- Horizon, dependencies, risks, lifecycle, evidence snippets
- Value type and finding type for sizing agent
- Execution complexity and timing dependency for roadmap agent
